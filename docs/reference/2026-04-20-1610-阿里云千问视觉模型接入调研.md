## 二、调用方式（OpenAI 兼容模式，推荐）

### 2.1 环境变量

```bash
DASHSCOPE_API_KEY=sk-xxx
```

### 2.2 Python 最小调用骨架

```python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.getenv("DASHSCOPE_API_KEY"),
    base_url="https://dashscope.aliyuncs.com/compatible-mode/v1",
)

completion = client.chat.completions.create(
    model="qwen3-vl-flash-2026-01-22",  # 或 gui-plus-2026-02-26
    messages=[
        {
            "role": "user",
            "content": [
                {"type": "image_url", "image_url": {"url": "https://.../xxx.png"}},
                {"type": "text", "text": "请定位..."},
            ],
        }
    ],
    extra_body={
        "vl_high_resolution_images": True,  # 高分辨率（建议截图类测试开启）
        "enable_thinking": False,           # 思考模式开关
    },
)

print(completion.choices[0].message.content)
# 思考内容（若 enable_thinking=True）
print(getattr(completion.choices[0].message, "reasoning_content", ""))
```

> **说明**：`enable_thinking` 通过 `extra_body` 传递（OpenAI SDK 没有此原生字段）。对 `qwen3-vl-flash` 和 `gui-plus-2026-02-26` 均适用。

### 2.3 图像输入格式

三种方式任选：
1. **公网 URL**：`{"type": "image_url", "image_url": {"url": "https://..."}}`
2. **Base64**：`{"type": "image_url", "image_url": {"url": "data:image/png;base64,<b64>"}}`
3. **本地路径**（DashScope 原生接口，不是 OpenAI 兼容）：`{"image": "file:///path/to.png"}`

**图像限制**：
- 格式：BMP / JPEG / PNG / TIFF / WEBP / HEIC
- 单图 ≤ 10 MB（Base64 编码后也要 ≤ 10 MB）
- 宽高均 > 10 px，宽高比 ≤ 200

## 三、GUI-Plus 专属约束（非常重要）

### 3.1 必须使用推荐的 System Prompt

官方明确：**不使用指定 System Prompt 会严重影响输出质量**。其内容包含：
- `<tools>` 块定义一个名为 `computer_use` 的 function，提供 `click / type / scroll / wait / terminate` 等动作枚举
- 固定的 Response 输出格式：`Action: <imperative>` + `<tool_call>{...json...}</tool_call>`

**输出格式示例**：
```
Action: Click the Firefox icon
<tool_call>
{"name": "computer_use", "arguments": {"action": "click", "coordinate": [120, 340]}}
</tool_call>
```

### 3.2 与本项目 `<point>x y</point>` 风格的不兼容之处

| 方面 | 本项目当前 | GUI-Plus |
|------|----------|----------|
| 坐标输出 | `<point>x y</point>` 0-1000 相对坐标 | `"coordinate": [x, y]` 像素绝对坐标 |
| 任务范式 | 回答「XX 在哪」 | 回答「下一步动作是什么」 |
| 无动作场景 | 直接定位 | 需要用 `action=answer` 包装 |

**接入策略建议**：
- 对纯定位任务（POS_001~003），**可以改用自定义 system prompt 直接问坐标**（不启用 computer_use tool），让 GUI-Plus 作为普通 VL 模型使用。需实验验证是否仍有优势。
- 或者接受 tool_call 格式，在 `test_runner._extract_predictions()` 增加 tool_call JSON 解析分支，从 `arguments.coordinate` 取出像素坐标，再按图片尺寸换算回 0-1000 统一比较。

### 3.3 推荐 extra_body

```python
extra_body={"vl_high_resolution_images": True}
```
开启后图像 Token 上限为 16384（约 12.9 MP 像素处理能力），UI 截图类任务更稳。

## 四、Qwen3-VL-Flash 能力点

来源：通义官方 + 百炼控制台介绍。

- **效果**：声称在小尺寸档位上"超过 qwen2.5-vl-72b"，效果优于开源版 Qwen3-VL-30B-A3B
- **核心能力**：
  - 空间感知、万物识别
  - 视觉 **2D / 3D 定位**（本项目用得上）
  - 长视频 / 长文档（百万上下文）
  - 文档 OCR、UI Agent 控制
- **思考模式切换**：同通用 Qwen3 系列，`extra_body={"enable_thinking": bool}`
- **响应字段**：思考内容在 `message.reasoning_content`（非流式）或 delta.reasoning_content（流式）

## 五、计费计算（图像 Token）

### 5.1 公式

```
image_tokens = (h_bar × w_bar) / (token_pixels) + 2
```

- `token_pixels`：**gui-plus 为 28×28=784**；**qwen3-vl-flash 为 32×32=1024**
- `h_bar`、`w_bar`：预处理缩放后宽高，要求是 `factor` 的整数倍（gui-plus factor=28；qwen3-vl factor=32）
- `+2`：`<vision_bos>` 和 `<vision_eos>` 两个标记

### 5.2 缩放规则（简化）

- **上限**：`vl_high_resolution_images=True` 时是 16384 token 对应的像素数；False 时是 `max_pixels` 参数
- **下限**：`min_pixels = 4 × factor × factor`
- 超过上限则等比缩小，低于下限则放大

### 5.3 对本项目 1280×640 截图的估算

转 gui-plus 模型为例（factor=28）：
- `h_bar = round(640/28)*28 = 23*28 = 644`
- `w_bar = round(1280/28)*28 = 46*28 = 1288`
- 面积 829,472 < 16384×784 上限，不缩放
- Token = 829472 / 784 + 2 ≈ **1,060**

### 7.2 `src/client.py` 改造

现状只支持 ARK 一个 base_url。需要：
- 按 `ModelConfig.provider` 选择 client 实例（`ark` / `dashscope`）
- 两家都是 OpenAI 兼容，可复用 OpenAI SDK，只切 `base_url` 和 `api_key`
- `enable_thinking` 的传递方式差异：
  - ARK: `extra_body={"thinking": {"type": "enabled"|"disabled"}}`（按现有实现）
  - DashScope: `extra_body={"enable_thinking": True|False}`

### 7.3 `<point>` 解析兼容

GUI-Plus 默认输出 **tool_call JSON** 格式，需要在 `_extract_predictions()` 增加对：

```text
<tool_call>{"name":"computer_use","arguments":{"action":"click","coordinate":[x,y]}}</tool_call>
```

的解析分支（从 `coordinate` 取像素绝对坐标），并**按图片尺寸换算为 0-1000 相对坐标**以统一评估。或者改用自定义提示词，跳过 computer_use tool。
